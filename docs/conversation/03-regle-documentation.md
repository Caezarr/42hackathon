# Moment 3 — Règle de documentation continue

Date : 18 juillet 2026

## Demande

Créer des fichiers Markdown à chaque moment important de la conversation.

## Convention adoptée

- Les notes sont placées dans `docs/conversation/`.
- Les fichiers sont numérotés chronologiquement.
- Une nouvelle note est créée pour chaque décision, changement d’architecture, étape d’implémentation ou résultat de test important.
- `00-INDEX.md` tient la liste chronologique des notes.
- Les autres projets présents dans le workspace ne sont ni lus ni utilisés comme inspiration.
- Les notes décrivent les décisions et leurs raisons, sans recopier inutilement toute la conversation.

## Prochain fichier prévu

Le prochain moment documenté sera le démarrage de l’implémentation et le choix définitif de la stack du CLI.
