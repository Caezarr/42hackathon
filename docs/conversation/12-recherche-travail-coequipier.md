Comparer les éléments poussés par un coéquipier sur GitHub avec la roadmap et l’architecture déjà documentées.

- branche par défaut : `main` ;
- seule branche exposée par le dépôt : `main` ;

## Conclusion

Aucun changement attribuable au coéquipier n’est visible dans le dépôt cible au moment de cette vérification.

Le travail peut se trouver :

- sur un fork personnel ;
- dans une branche locale non poussée ;
- dans un autre dépôt ;
- dans une branche dont aucune pull request n’a encore été ouverte.

## Information nécessaire pour la comparaison

L’un des éléments suivants suffit :

- URL du fork ;
- URL de la branche ;
- URL de la pull request ;
- nom GitHub du coéquipier ;
- hash du commit.

Une fois le travail localisé, la comparaison devra couvrir :

1. adéquation avec le plugin Codex ;
2. exécution réellement locale ;
3. licences open source ;
4. capacité à appeler un vrai numéro public ;
5. cohérence avec le serveur MCP ;
6. conflits ou éléments réutilisables ;
7. écarts par rapport au MVP de six heures.
